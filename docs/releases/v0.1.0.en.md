# MeshFree v0.1.0 Release Notes

## 1. Release Position

`v0.1.0` is the first formal `MVP` milestone release of `MeshFree`.

This release is not intended to be a large production platform yet. Its purpose is to make the three core business loops truly work end to end and to establish a version that is:

- runnable
- demo-ready
- deployable
- maintainable

## 2. Included in This Release

This release already includes the following core capabilities:

- Public browsing of approved model resources
- Public model detail page
- Public ZIP download for approved resources
- Public submission of new model resources
- Single-admin login
- Admin submission list and detail view
- Admin approval, rejection, and deletion actions

## 3. Supporting Docs and Project Assets

The repository already includes the following documentation and project support materials:

- Chinese and English MVP specification documents
- Chinese and English API design documents
- Chinese and English deployment guides
- Chinese server maintenance guide
- Admin maintenance script
- Example deployment configuration for `Nginx` and `PM2`

## 4. Explicitly Out of Scope

To keep the `MVP` scope controlled, this release does not include:

- Normal user registration and login
- Search
- Category filters
- Comments
- Favorites
- Likes
- 3D online preview
- Object storage
- Docker deployment
- Multi-admin support

## 5. Tech Stack

- Frontend: `React + TypeScript + Vite`
- Backend: `Node.js + Express + TypeScript`
- Database: `SQLite`
- ORM: `Prisma`
- Process manager: `PM2`
- Reverse proxy: `Nginx`
- File storage: server local disk

## 6. Current Deployment Status

As of `v0.1.0`:

- Real server deployment has been completed
- The primary domain is already connected to the current deployment setup
- Production `CORS` has been tightened
- The admin login endpoint has basic rate limiting
- The public submission endpoint has basic rate limiting
- Current basic verification has not revealed obvious problems

## 7. Known Limitations

This release still has the following limitations:

- Uploaded files are stored on the server local disk instead of object storage
- The focus is still the MVP core loop, not a full community platform
- There is no user account system, so submission follow-up depends on the contact field
- There is no search, category system, or analytics feature yet

After `v0.1.0`, the project priority should focus more on:

- continued online environment verification
- documenting deployment and maintenance lessons
- targeted fixes when small problems appear
- considering the next round of readability, UI, or feature improvements after the version remains stable
