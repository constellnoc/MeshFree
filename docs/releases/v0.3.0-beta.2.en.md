# MeshFree v0.3.0-beta.2 Release Notes

## 1. Release Position

`v0.3.0-beta.2` is a workflow and governance beta for `MeshFree`.

This version is focused less on adding a brand-new public showcase feature and more on making the current upload-review-publish loop more operable, more maintainable, and easier to test locally.

## 2. Summary

This beta focuses on:

- admin-side tag governance completion
- multi-tag filtering
- Chinese copy normalization
- review workflow usability improvements

The main goal of this release is to move the tag system from "structured in data" to "operable in admin UI".

## 3. Added in This Release

- Admin APIs for governed raw-tag handling
  - ignore a private custom tag
  - resolve a private custom tag to an existing public tag
  - create a new public tag directly from a private custom tag
  - create a new public tag directly from the admin panel
- Admin-side inline forms for:
  - creating a new governed public tag
  - binding a raw custom tag to an existing public tag
- Interactive raw-tag cards in the admin dashboard
- Bilingual validation flow for admin-created tags:
  - `slug`
  - English name
  - Chinese name
  - `scopeLevel`
- Localized admin feedback messages for the new tag-governance actions

## 4. Changed in This Release

- Changed the admin dashboard from read-only raw-tag display to actionable raw-tag governance flow
- Changed public tag selection UX in the admin dashboard so selected tags move to a separate active row
- Changed homepage tag filtering from single-select to multi-select
- Changed multi-tag result summary copy to match the new filtering behavior
- Changed Chinese terminology consistency across the current UI surface, including:
  - `Gallery` -> `模型库`
  - `Dashboard` -> `管理面板`
  - `Admin Dashboard` -> `管理员管理面板`
- Changed local admin defaults for development convenience:
  - username: `local`
  - password: `123`

## 5. Current Tag Governance Behavior

In this beta:

- public models bind only to governed public tags
- user-submitted custom tags remain private by default
- private custom tags do not become public automatically
- admins can explicitly decide whether a private custom tag should:
  - be ignored
  - map to an existing public tag
  - become a new public tag
- newly created governed tags require both English and Chinese display names
- governed tag actions now update the submission state directly in the admin workflow

## 6. Validation Status

Verified in this release:

- `server` build passes
- `client` build passes
- lint checks for the files touched in this release pass

Note:

- the repository still contains some older frontend lint issues outside the scope of this release

## 7. Known Gaps

- admin flow for converting private custom tags into aliases is still not implemented
- admin tag governance UI can still be further modularized
- full multilingual UI switching still needs more end-to-end polishing
- viewer bundle size is still relatively large
