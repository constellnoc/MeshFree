# MeshFree MVP 需求说明

## 1. 项目概述

`MeshFree` 是一个用于 MVP 演示的轻量模型上传与下载网站。

本项目的核心目标如下：

- 允许游客浏览并下载已审核通过的模型资源
- 允许游客提交模型资源并等待管理员审核
- 只允许一个管理员账号登录后台进行审核与管理
- 第一版以“能跑起来、能演示、易部署”为优先目标

## 2. 目标用户

当前 MVP 主要服务以下三类实际使用者：

- 浏览并下载资源的游客
- 提交模型资源等待审核的投稿者
- 唯一负责审核和管理的管理员

## 3. 角色与权限

### 游客

游客可以：

- 查看已审核通过的模型列表
- 查看模型详情页
- 下载已审核通过的 ZIP 模型文件
- 提交新的模型资源

游客不可以：

- 注册账号
- 以普通用户身份登录
- 访问后台页面
- 审核或删除投稿

### 管理员

管理员可以：

- 使用账号密码登录后台
- 查看全部投稿记录
- 审核待处理投稿
- 通过投稿
- 拒绝投稿并填写拒绝原因
- 删除投稿及其关联文件

## 4. MVP 功能范围

### 包含的功能

- 前台首页
- 模型详情页
- 投稿页面
- 公开 ZIP 下载
- 管理员登录
- 管理员审核页面
- 投稿状态管理

### 第一版暂不包含的功能

- 用户注册
- 普通用户登录
- 搜索
- 分类筛选
- 评论
- 收藏
- 点赞
- 3D 在线预览
- 通知系统
- 批量上传
- Docker 部署
- 对象存储
- 防盗链
- 大文件断点续传

## 5. 资源规则

### 模型文件

- 仅允许 `.zip` 文件
- 最大文件大小为 `20MB`
- 文件存储在服务器本地磁盘

### 封面图

- 封面图为必填项
- 支持格式：`jpg`、`jpeg`、`png`、`webp`
- 最大文件大小为 `2MB`
- 文件存储在服务器本地磁盘

### 投稿字段

每条投稿包含以下信息：

- 标题
- 简介
- 联系方式
- 封面图
- ZIP 文件
- 审核状态
- 拒绝原因
- 创建时间
- 审核时间

## 6. 联系方式字段

投稿表单必须包含联系方式字段：

`Contact info (QQ / WeChat / Email)`

由于 MVP 不包含用户账号系统，因此管理员需要通过该字段联系投稿者。

## 7. 投稿状态

每条投稿具有以下三种状态之一：

- `pending`：待审核
- `approved`：审核通过并对游客可见
- `rejected`：审核拒绝并对游客隐藏

## 8. 审核规则

### 审核通过

当投稿被通过时：

- 状态更新为 `approved`
- 资源在前台页面可见
- 资源可被游客下载

### 审核拒绝

当投稿被拒绝时：

- 状态更新为 `rejected`
- 数据仍保留在数据库中
- 必须保存拒绝原因

## 9. 前台展示规则

首页仅展示已审核通过的资源。

每个资源卡片只展示：

- 封面图
- 标题
- 简介摘要

首页不展示：

- 下载次数
- 分类
- 标签
- 作者主页

## 10. 下载规则

- 下载不需要登录
- 仅 `approved` 状态的资源允许下载
- 文件下载通过后端 API 提供
- 第一版不实现防盗链

## 11. 管理员规则

- 系统中仅存在一个管理员账号
- 不开放公开注册
- 管理员通过账号密码认证
- 后台页面必须受保护
- 所有审核操作都必须通过管理员认证

## 12. 文件删除策略

当管理员删除一条投稿时，需要同时执行以下操作：

- 删除数据库记录
- 删除对应的封面图文件
- 删除对应的 ZIP 文件

该策略可以保证数据库与本地文件存储保持一致。

## 13. 推荐技术栈

- 前端：`React + TypeScript + Vite`
- 后端：`Node.js + Express + TypeScript`
- 数据库：`SQLite`
- ORM：`Prisma`
- 进程管理：`PM2`
- 反向代理：`Nginx`
- 部署系统：`Ubuntu`
- 文件存储：本地磁盘

## 14. 部署约束

当前服务器资源为：

- `2 核 CPU`
- `2GB 内存`
- `40GB SSD`
- `200GB/月流量`
- `3Mbps 带宽`

基于这些限制，MVP 需要遵循以下原则：

- 严格限制上传文件大小
- 使用本地文件存储
- 避免复杂基础设施
- 以稳定性优先于高级功能

### 公开部署基础约束

在当前 MVP 阶段，公开部署还应遵循以下基础约束：

- 前后端优先通过同域名部署，由 `Nginx` 统一提供入口
- 生产环境应限制 `CORS` 来源，只允许正式站点域名访问后端 API
- 管理员登录接口应具备基础限流
- 公开投稿接口应具备基础限流

这些约束不属于“扩展功能”，而是为了让当前 MVP 在公网环境中更稳妥地运行。

## 15. 产品目标

本 MVP 的目标不是构建大型生产平台，而是完成一个：

- 可以运行
- 可以演示
- 便于答辩说明
- 结构合理
- 便于维护

的比赛项目。
