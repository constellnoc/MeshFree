## 3. 今日处理的前台问题

### 3.1 投稿成功反馈改为弹窗

原问题：

- 投稿成功后的反馈只显示在页面下方
- 用户写完一大段内容后，不容易第一时间看到成功信息

本轮改动：

- 将成功反馈从页面底部提示改为居中弹窗
- 弹窗内保留：
  - 成功提示
  - 投稿 ID
  - 当前状态
  - 关闭入口
- 支持点击遮罩关闭
- 支持按 `Esc` 关闭

### 3.2 首页隐藏简介

原问题：

- 首页模型卡片会直接铺开简介
- 卡片高度被拉长，影响扫读效率

本轮改动：

- 首页卡片不再显示模型简介
- 首页保留：
  - 封面
  - 标题
  - 标签
  - 详情入口
  - 创建时间

### 3.3 模型详情简介保留换行

原问题：

- 简介中的普通回车在详情页显示时会被折叠
- 用户必须空一行，视觉上才像换段

本轮改动：

- 模型详情页的简介显示改为保留原始换行
- 通过样式让普通回车也能正确显示为换行

结果：

- 现在简介的源文本结构在详情页里更接近用户实际输入

### 3.4 预览器支持鼠标中键拖动

原问题：

- 预览器只能旋转和缩放
- 无法通过鼠标中键平移视角

本轮改动：

- 开启 `OrbitControls` 的平移能力
- 显式把鼠标中键映射为平移
- 同步更新底部操作提示文案

结果：

- 现在预览器支持：
  - 左键拖拽旋转
  - 中键拖拽平移
  - 滚轮缩放

## 4. 今日确认的资源包规则

这轮也顺手确认了一个实际资源整理问题：

- 外层上传格式仍然只支持 `ZIP`
- 不支持 ZIP 内再套 `RAR`
- 如果 `source` 文件夹里只有一个 `RAR`，系统当前不会继续打开它

当前正确整理方式应为：

- 先把 `RAR` 在本地解开
- 再把真实模型文件与贴图一起打进最终上传的 `ZIP`

同时确认：

- `OBJ` 本身允许由多个配套文件组成
  - `obj`
  - `mtl`
  - 贴图
- 当前真正的限制不是“文件多”，而是：
  - 不能出现多个候选主模型文件，否则系统不会自动决定预览哪个
