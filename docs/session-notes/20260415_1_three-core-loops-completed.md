## 4. 本轮遇到并解决的问题

### 4.1 上传路径逻辑容易重复

随着闭环一、闭环二、闭环三陆续接入真实业务后，公开浏览、公开投稿、后台删除都需要处理上传路径和本地文件。

为避免：

- 路径规则分散
- URL 拼接方式不一致
- 删除文件时路径解析出错

本轮将这部分逻辑抽到了：

- `server/src/lib/uploads.ts`

这样后续继续维护时，只需要维护一套上传路径约定。

### 4.2 管理员登录验证需要真实密码校验

原本后台登录只是 placeholder，直接签发 token，并没有真正读取数据库管理员记录。

本轮已补齐：

- `Prisma` 查询管理员账号
- `bcryptjs.compare()` 校验密码
- JWT 中明确写入管理员角色

这样后台接口才真正具备“只有管理员可访问”的含义。

### 4.3 本地管理员验证时存在开发态密码问题

为了完成“真实登录”验证，本轮对本地开发数据库中的 `admin` 账号重新执行了 seed。

需要特别说明：

- 这一步只影响本地开发数据库
- 没有把真实密码写进仓库
- 在进入部署阶段前，应重新明确本地开发密码和生产环境密码策略
