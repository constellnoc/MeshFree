# MeshFree 首页深化、弹窗详情与滚动动效总结

## 1. 本轮主题

本轮工作的主题，是在 `20260418_2` 已经明确首页 Hero / 几何元素 / Gallery 一体化方向之后，继续把这套前端体验实际落地，并围绕用户即时反馈进行多轮收敛。

本轮实际聚焦的内容包括：

- 首页 Hero 视觉落地
- 几何图形背景与滚动响应调整
- 顶部导航均匀化调整
- `Submit` 到 `Upload` 的命名与路由统一
- 模型详情从独立页面改为大弹窗方案
- 弹窗细节继续收口

截至本轮结束，项目重点已经从“讨论首页方向”推进到“首页与 Gallery 一体化体验已初步落地，可继续迭代细节”。

## 2. 本轮已完成内容

### 2.1 首页 Hero 与 Gallery 一体化结构已落地

本轮已在首页实际完成：

- 将原先简单说明区改为 Hero 区
- Hero 与 Gallery 保持单页连续结构
- 顶部 `Gallery` 仍然滚动到模型区
- 页面标题与导航高亮不再只依赖点击，而会根据首页实际滚动区块联动

当前首页已经不是“一个说明卡片 + 一个模型列表”，而是开始具备明确的上半屏表达与下半屏浏览节奏。

### 2.2 Hero 改为无框背景几何系统

本轮在 Hero 视觉上已经完成以下调整：

- 去掉之前不符合预期的大白框 Hero 容器
- 去掉临时加入的额外英文过渡文案
- 让主标题与副文案直接浮在背景上
- 使用红 / 黄 / 蓝几何图形作为背景主视觉
- 几何图形加入清晰与模糊层级
- 图形在滚动时产生位移、旋转、缩放变化
- 进入模型区前加入更快的退场曲线

当前效果仍可继续微调，但方向已经从“卡片式 Hero”修正到“背景式几何视觉层”。

### 2.3 顶部导航进一步调整为更均匀的节奏

本轮继续收紧并调整了顶栏布局：

- 导航不再保持明显的左中右割裂感
- `Gallery / Search / About / Sign in / Upload` 收进同一条导航轨道
- 导航项改为统一列宽分配
- `Upload` 保持轻度强调，但不再破坏整体分隔节奏

这意味着顶栏已经更接近之前讨论中的“均匀分隔”目标。

### 2.4 用户可见的 `Submit` 已统一为 `Upload`

本轮已完成：

- 用户可见文案中的 `Submit` 统一改为 `Upload`
- 投稿页主标题、按钮、说明文案同步改名
- 主路由调整为 `/upload`
- 保留 `/submit -> /upload` 的兼容跳转

这解决了当前前端界面里“Upload / Submit 混用”的命名不统一问题。

### 2.5 模型详情已改为大弹窗方案

本轮已将模型详情从“整页跳转”调整为“首页上方叠加大弹窗”：

- 在 Gallery 点击模型卡片时，优先打开弹窗详情
- 背景保留当前首页与 Gallery 浏览位置
- 背景加入暗层与毛玻璃效果
- 支持点击遮罩关闭
- 支持 `Esc` 关闭
- 如果用户直接访问 `/models/:id`，仍然保留独立详情页兜底

这样做之后，用户从 Gallery 浏览模型再返回时，不会再被强制打回首页顶部。

### 2.6 弹窗细节已进行第一轮收口

本轮围绕弹窗观感继续做了几次小调整：

- 去掉弹窗里的文字 `Close` 按钮
- 改为右上角关闭叉号
- 将关闭图标缩小并贴近右上角
- 去掉关闭按钮不必要的白底白边
- 调整弹窗容器上下留白，使上下边距更均匀
- 当前弹窗整体不再作为大滚动容器
- 若简介过长，只让简介区域单独滚动

虽然视觉上还可继续精修，但交互结构已经稳定。

## 4. 当前已确认的实现细节

### 4.1 标题与导航高亮规则

当前首页内部已改为基于滚动区块联动：

- Hero 区：`MeshFree-Home`
- Gallery 区：`MeshFree-Gallery`
- Footer / About 区：`MeshFree-About`

这意味着标题和高亮不再只反映“点击过什么”，而开始反映“当前浏览到哪里”。

### 4.2 几何滚动速度的主要调节点

当前如需继续调整几何图形的滚动体感，主要看两个位置：

1. `client/src/pages/HomePage.tsx`
   - `window.scrollY / (viewportHeight * 0.66)`
   - `const exitProgress = Math.max((heroScrollProgress - 0.48) / 0.26, 0);`

2. `client/src/index.css`
   - `geometry-shape-red-haze`
   - `geometry-shape-yellow-orb`
   - `geometry-shape-blue-disc`
   - `geometry-shape-blue-ring`
   - `geometry-shape-red-bar`
   - `geometry-shape-yellow-chip`

前者控制“整体动画推进速度与退场时机”，后者控制“每个图形具体移动与旋转多少”。

### 4.3 弹窗路由实现方式

当前弹窗详情采用的是：

- 背景路由 `backgroundLocation`
- 叠加路由渲染 modal
- 直接访问模型链接时保留独立详情页

这套方案的好处是：

- Gallery 浏览位置天然保留
- 分享模型详情链接仍然可用
- 不需要额外维护复杂的全局状态机

## 5. 当前仍可继续优化的点

本轮虽然已经完成主要结构落地，但以下内容仍然存在继续优化空间：

- Hero 几何图形的构图、运动节奏还可以继续收敛
- 顶栏虽然更均匀，但还可以继续打磨视觉重量
- 弹窗当前已可用，但整体审美仍可继续精修
- 模型详情内容布局还不够像真正的预览器

目前这些问题更偏“产品打磨”，而不是“功能缺失”。

## 6. 当前暂未实现的方向

本轮讨论过但尚未正式落地的内容包括：

- 类似 `Sketchfab` 的真实 3D 模型拖拽预览
- 更像预览器的大图左侧 / 信息右侧布局深化
- 利用弹窗继续承载未来 3D viewer

也就是说，本轮已经先把“交互壳子”搭好，但还没有真正进入 3D viewer 能力建设。

## 7. 当前阶段结论

截至 `20260418_3`，项目已经具备：

- 固定顶栏与统一命名的 `Upload` 入口
- Hero / Gallery 单页连续结构
- 基于背景几何的首页第一屏
- 随滚动联动的标题与高亮
- Gallery 上方的大弹窗模型详情
- 保留独立详情页兜底的路由结构

这意味着当前前端已经从：

- 纯功能型 MVP 页面

推进到：

- 开始具备产品浏览体验和展示节奏的 MVP 前端

## 8. 下一步建议

下一步建议优先考虑以下方向：

- 继续收紧弹窗审美，而不是大改交互结构
- 将弹窗布局进一步向“预览器”形态靠近
- 若正式进入 3D 预览阶段，优先先做详情弹窗内的 viewer，而不是先改上传流程
- 继续通过小步微调的方式调整首页几何图形，不要一次性把动效做重
