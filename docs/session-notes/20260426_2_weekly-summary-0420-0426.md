## 2. 今日确认并落地的规则

今天按已确认方案，正式落地了这些规则：

- 私有自定义标签不会自动公开
- 管理员可以对每个私有自定义标签执行三类操作：
  - 绑定到已有公开标签
  - 转成新的公开标签
  - 忽略
- 新建公开标签时，必须同时填写：
  - `slug`
  - 英文名
  - 中文名
  - `scopeLevel`
- 从私有自定义标签新建公开标签后，该标签会同时绑定到当前投稿

一句话：

- 用户负责提供原始标签文本，管理员负责决定它是否以及如何进入公开标签系统。

## 3. 今日已完成的实现

### 3.1 后端接口

已在管理端路由中补齐最小接口集：

- `POST /api/admin/tags`
  - 直接新建公开标签
- `PATCH /api/admin/raw-tags/:id/ignore`
  - 忽略私有自定义标签
- `PATCH /api/admin/raw-tags/:id/resolve-existing`
  - 绑定到已有公开标签
- `POST /api/admin/raw-tags/:id/create-tag`
  - 从私有自定义标签直接创建公开标签

补充处理：

- 新建标签时校验 `slug`、中英文名、`scopeLevel`
- 处理完成后直接返回更新后的投稿详情，方便前端局部刷新

### 3.2 管理员页 UI

已在管理员页完成对应交互：

- 私有自定义标签从只读 chip 改为可操作卡片
- 每张卡片可以直接执行：

  - 新建自有标签
  - 忽略
- 在公开标签区域新增“新建自有标签”表单
- 新建和绑定动作都支持在当前页面内直接完成，不再只是讨论方案
