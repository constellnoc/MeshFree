# MeshFree 多源格式转 GLB 预览工作文档

最后更新：2026.04.27

## 1. 文档定位

本文档只讨论一个明确范围内的目标：

- 支持更多源格式模型资源上传与下载
- 上传资源仍然只接受 `ZIP`
- 服务器尝试把源格式模型转换为 `glb`
- 在线预览统一使用转换后的 `glb`
- 即使转换失败，资源仍允许上传、审核、公开和下载

一句话：

- 源格式用于上传和下载，`glb` 只用于在线预览。

## 2. 本轮已确认规则

### 2.1 格式优先级

当前优先级：

1. `OBJ`
2. `FBX`
3. `DAE`
4. `BLEND`

### 2.2 各格式支持边界

当前统一规则：

- 允许上传
- 允许下载源文件
- 允许尝试转换 `glb` 预览
- 转换失败后仍允许保留资源

适用格式：

- `OBJ`
- `FBX`
- `DAE`
- `BLEND`

### 2.3 ZIP 规则

已确认：

- 上传仍然只接受 `ZIP`
- 允许 ZIP 内包含子目录
- 不支持一个 ZIP 内同时存在多个候选主模型文件

如果检测到多个候选主模型文件：

- 允许资源上传
- 不自动生成预览
- 记录说明，类似：
  - `资源包内包含多个不相关的候选模型文件，当前不支持自动选择主模型预览。`

说明：

- “允许子目录”表示 ZIP 内文件可以位于嵌套目录中，而不要求所有文件平铺在根目录。

### 2.4 转换失败时的产品行为

已确认：

- 转换失败允许入库
- 转换失败允许公开展示
- 转换失败不影响下载
- 普通访客不显示技术失败原因
- 管理员可以看到失败原因
- 用户可以看到失败原因

访客侧建议统一提示：

- `该资源当前暂无在线预览，可下载源文件查看。`

### 2.5 预览边界

当前阶段正式定义为：

- 优先保证静态模型的基础在线预览
- 动画优先级较低，但未来要做
- 骨骼优先级较低，但未来要做
- 当前阶段对动画 / 骨骼只做兼容尝试，不承诺稳定效果

贴图与材质规则：

- 贴图缺失时，允许尝试生成不完整预览
- 材质异常时，允许尝试生成预览
- 是否对访客开放该预览，由管理员人工判断
- 如果管理员认为预览不合理，可关闭该模型的在线预览功能

### 2.6 转换文件存储

已确认：

- 服务器保存一个仅用于在线预览的 `glb`
- 用户不能直接访问和下载该预览 `glb`
- 原始 ZIP 继续保留
- 删除资源时，同时删除对应 `glb`
- 更新资源时，旧 `glb` 删除

### 2.7 审核与公开字段

审核状态与公开状态是两类字段：

第一类：审核状态

- `pending`
- `approved`
- `rejected`

第二类：公开状态

- 是否公开给访客

说明：

- `approved` 不等于一定公开
- 模型即使已审核通过，也可以被管理员手动关闭公开
- 未公开模型对访客强拦截，不可见

### 2.8 预览开关

管理员可以独立控制某模型的在线预览是否开放。

如果关闭预览：

- 访客仍可看到：
  - 详情
  - 封面
  - 标签
  - 下载入口
- 访客不可使用在线预览
- 前台显示注脚，类似：
  - `该资源的在线预览已被管理员关闭。`

### 2.9 访客强拦截规则

如果模型被设置为“不公开”，则对访客执行强拦截：

- 首页列表不可见
- 搜索不可见
- 详情页不可见
- 下载不可用

一句话：

- 不公开就是对访客完全不可见。

## 3. 当前阶段不做什么

本轮明确不做：

- 浏览器原生直接预览 `OBJ / FBX / DAE / BLEND`
- 一个 ZIP 内多个候选主模型文件的人工选择器
- 动画稳定支持承诺
- 骨骼稳定支持承诺
- 自动材质修复
- 自动质量评分
- 复杂异步任务系统
- `UE / Unity` 导入链路
- 其他新格式扩展如 `USD / ABC / STL`

## 4. 建议新增的核心状态字段

### 4.1 资源源格式

- `sourceFormat`
  - `obj`
  - `fbx`
  - `dae`
  - `blend`
  - `glb`
  - `unknown`

### 4.2 系统转换状态

- `previewConversionStatus`
  - `not_attempted`
  - `success`
  - `warning`
  - `failed`

### 4.3 系统转换说明

- `previewConversionMessage`
  - 保存技术原因或简化说明
  - 例如：
    - 缺少贴图
    - 检测到多个候选主模型文件
    - 格式转换失败
    - 解析成功但材质不完整

### 4.4 管理员预览开关

- `isPreviewEnabled`
  - `true`
  - `false`

### 4.5 公开开关

- `isPublicVisible`
  - `true`
  - `false`

### 4.6 可选补充字段

- `previewReviewNote`
- `hasMissingTextures`
- `hasPreviewWarnings`

## 5. 管理员功能目标

管理员界面后续需要支持以下能力：

### 5.1 查看模型格式与预览状态

至少显示：

- 源格式
- 转换状态
- 转换失败原因
- 是否缺贴图
- 是否公开
- 是否开放在线预览

### 5.2 控制公开状态

管理员可以手动：

- 打开公开
- 关闭公开

关闭公开后：

- 访客完全不可见

### 5.3 控制预览状态

管理员可以手动：

- 打开预览
- 关闭预览

关闭预览后：

- 资源仍可公开
- 详情与下载仍保留
- 仅在线预览不可用

### 5.4 人工审核预览合理性

管理员需要从多个角度人工查看预览是否合理。

如果预览不合理：

- 可关闭在线预览
- 可记录原因
- 可反馈用户修改

## 6. 前台行为目标

### 6.1 已公开且预览可用

访客可看到：

- 资源卡片
- 详情页
- 下载
- 在线预览

### 6.2 已公开但预览不可用

访客可看到：

- 资源卡片
- 详情页
- 下载
- 友好提示

不显示技术失败原因。

建议提示：

- `该资源当前暂无在线预览，可下载源文件查看。`
- 或
- `该资源的在线预览已被管理员关闭。`

### 6.3 未公开

访客不可看到：

- 列表
- 搜索结果
- 详情页
- 下载

## 7. 分步实施原则

本任务必须拆分实施，每一步都要满足：

- 范围小
- 可独立验证
- 不顺手扩范围
- 每一步都能单独提交 commit
- 每一步完成后再讨论下一步

原则：

- 先改“状态与结构”
- 再改“上传检测”
- 再改“转换与存储”
- 再改“管理员控制界面”
- 最后改“访客显示逻辑”

## 8. 分步实施计划

### Step 1：补状态字段与后端基础结构

#### 目标

先把数据库和后端数据结构补齐，为后续多格式检测、预览状态、公开控制做准备。

#### 要做的事

- 给资源 / 投稿模型补：
  - `sourceFormat`
  - `previewConversionStatus`
  - `previewConversionMessage`
  - `isPreviewEnabled`
  - `isPublicVisible`
  - 可选：`hasMissingTextures`
- 明确默认值
- 调整后端类型、返回结构、管理端详情接口

#### 本步不做

- 不做真实格式转换
- 不做上传解析逻辑修改
- 不做前端完整 UI
- 不做访客侧逻辑切换

#### 验收标准

- 后端能返回这些新字段
- 管理员详情接口能看到这些字段
- 旧流程不被破坏

#### 建议 commit

- `feat: add preview and visibility status fields`

### Step 2：补 ZIP 扫描与源格式检测

#### 目标

让系统能在 ZIP 内识别主模型候选、判断源格式，并处理“多个候选文件”的情况。

#### 要做的事

- 允许扫描子目录
- 支持识别：
  - `.obj`
  - `.fbx`
  - `.dae`
  - `.blend`
- 检查候选主模型文件数量
- 如果只有一个候选：
  - 记录 `sourceFormat`
- 如果多个候选：
  - 不阻止上传
  - 不自动预览
  - 写入说明：
    - `检测到多个不相关的候选模型文件，当前不支持自动选择主模型。`

#### 本步不做

- 不做真实转换
- 不做前台提示
- 不做管理员按钮

#### 验收标准

- ZIP 扫描结果稳定
- 多候选文件时资源仍能上传
- 管理端可看到说明

#### 建议 commit

- `feat: detect source model format from uploaded zip`

### Step 3：实现服务端转换为 glb 的最小链路

#### 目标

让服务器能基于单一候选主模型文件尝试生成用于预览的 `glb`。

#### 要做的事

- 设计统一转换入口
- 为不同格式接入转换策略
- 成功时保存预览 `glb`
- 失败时记录失败原因
- 缺贴图 / 材质异常时记录 warning
- 删除资源或更新资源时清理旧 `glb`

#### 本步不做

- 不做管理员预览开关 UI
- 不做访客侧可见性切换
- 不承诺动画 / 骨骼稳定效果

#### 验收标准

- 成功资源可生成 `glb`
- 失败资源不影响上传与下载
- 数据库里可区分 `success / warning / failed`

#### 建议 commit

- `feat: add server-side source-to-glb preview conversion`

### Step 4：管理员界面补状态展示与控制开关

#### 目标

让管理员可以看到格式与转换状态，并主动控制公开和预览。

#### 要做的事

- 在管理员详情页显示：
  - 源格式
  - 转换状态
  - 失败原因
  - 贴图缺失信息
  - 公开状态
  - 预览状态
- 增加两个独立开关：
  - 是否公开
  - 是否开放在线预览
- 对关闭状态做明显视觉区分，例如灰色状态块
- 对关闭预览增加管理员注释入口或说明文案

#### 本步不做

- 不做访客侧最终强拦截
- 不做新一轮大范围样式重构

#### 验收标准

- 管理员能独立切换公开 / 预览
- 管理员能清楚看到转换问题
- 界面可读，不混淆审核状态与公开状态

#### 建议 commit

- `feat: add admin controls for public visibility and preview`

### Step 5：访客侧可见性与提示逻辑落地

#### 目标

把管理员设置真正落实到访客端。

#### 要做的事

- 已关闭公开：
  - 列表隐藏
  - 搜索隐藏
  - 详情不可访问
  - 下载不可访问
- 已公开但关闭预览：
  - 详情仍可访问
  - 下载仍可使用
  - 显示提示：
    - `该资源的在线预览已被管理员关闭。`
- 转换失败但公开：
  - 显示提示：
    - `该资源当前暂无在线预览，可下载源文件查看。`

#### 本步不做

- 不继续顺手做格式扩展 UI
- 不做复杂的用户通知系统

#### 验收标准

- 访客强拦截逻辑正确
- 预览关闭逻辑正确
- 提示文案准确

#### 建议 commit

- `feat: enforce public visibility and preview access rules`

### Step 6：补错误提示、反馈和手动验证清单

#### 目标

给这条链路补最基本的可维护性。

#### 要做的事

- 补用户侧与管理员侧提示文案
- 补手动测试清单
- 补文档：
  - `README`
  - API
  - release note
  - session note

#### 验收标准

- 本轮功能有明确可回归的检查路径
- 文档与代码状态一致

#### 建议 commit

- `docs: document multi-format preview workflow`

## 9. 手动测试优先顺序

后续每一步都优先按这些场景做手测：

1. ZIP 内单一 `OBJ`，带完整贴图
2. ZIP 内单一 `OBJ`，缺失贴图
3. ZIP 内单一 `FBX`
4. ZIP 内多个候选主模型文件
5. 转换失败但资源仍公开下载
6. 管理员关闭公开后访客强拦截
7. 管理员关闭预览后访客仍能下载
8. 删除资源时预览 `glb` 一并删除
9. 更新资源时旧 `glb` 被清理

## 10. 当前阶段结论

这轮功能的本质不是“浏览器原生支持更多格式”，而是：

- 扩展源格式输入能力
- 统一在线预览格式
- 把预览可用性从“技术问题”变成“系统状态 + 管理员裁定”

一句话：

- 系统负责尽量转，管理员负责决定要不要给访客看。
